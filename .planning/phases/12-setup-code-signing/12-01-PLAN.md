  <files>src-tauri/tauri.conf.json, .github/workflows/build.yml</files>
  <action>
**1. Update `src-tauri/tauri.conf.json` — add signCommand to bundle.windows:**

Replace the empty `"windows": {}` in the `bundle` section with:

```json
"windows": {
  "signCommand": "trusted-signing-cli -e https://wus2.codesigning.azure.net -d LeadrScribe %1"
}
```

Notes:
- `-e` is the Azure Trusted Signing endpoint URL. `wus2` = West US 2 region. The developer MUST update this to match their actual Azure Trusted Signing account region. Common endpoints:
  - `https://eus.codesigning.azure.net` (East US)
  - `https://wus2.codesigning.azure.net` (West US 2)
  - `https://weu.codesigning.azure.net` (West Europe)
  - `https://neu.codesigning.azure.net` (North Europe)
- `-d LeadrScribe` sets the description shown in UAC prompts when running the installer
- `%1` is a Tauri placeholder replaced with the file path to sign — it MUST be a standalone argument (not embedded in another arg) per Tauri issue #11754
- The `-a` (account) and `-c` (certificate profile) flags are intentionally omitted — trusted-signing-cli reads them from `AZURE_TRUSTED_SIGNING_ACCOUNT_NAME` and `AZURE_CERTIFICATE_PROFILE_NAME` env vars set in CI
- Authentication uses `AZURE_CLIENT_ID`, `AZURE_CLIENT_SECRET`, `AZURE_TENANT_ID` env vars (already passed in build.yml)

**2. Update `.github/workflows/build.yml` — add missing Azure Trusted Signing env vars:**

In the "Build with Tauri" step, add these env vars alongside the existing `AZURE_CLIENT_ID`, `AZURE_CLIENT_SECRET`, `AZURE_TENANT_ID`:

```yaml
AZURE_TRUSTED_SIGNING_ACCOUNT_NAME: ${{ inputs.sign-binaries && secrets.AZURE_TRUSTED_SIGNING_ACCOUNT_NAME || '' }}
AZURE_CERTIFICATE_PROFILE_NAME: ${{ inputs.sign-binaries && secrets.AZURE_CERTIFICATE_PROFILE_NAME || '' }}
```

These should be placed right after the existing `AZURE_TENANT_ID` line for grouping clarity.

**Important:** Do NOT modify any other env vars or workflow steps. The `trusted-signing-cli` installation step already exists. The `AZURE_CLIENT_ID`, `AZURE_CLIENT_SECRET`, `AZURE_TENANT_ID` are already passed. Only the account name and certificate profile name are missing.

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Set up Azure Trusted Signing resources and GitHub secrets</name>
  <action>
The developer must complete Azure and GitHub setup before code signing will work in CI.

**Azure Setup (one-time):**

1. **Create Trusted Signing account** in Azure Portal:
   - Search "Trusted Signing" -> Create
   - Choose a region (the endpoint URL in tauri.conf.json must match this region)
   - Note the account name

2. **Create certificate profile** in the Trusted Signing account:
   - Go to your Trusted Signing account -> Certificate profiles -> Add
   - Choose "Public Trust" profile type for code signing
   - Note the profile name

3. **Create App Registration** for CI authentication:
   - Azure Portal -> App registrations -> New registration
   - Create a client secret under Certificates & secrets
   - Note: Application (client) ID, Client secret value, Directory (tenant) ID

4. **Grant signing permission** to the App Registration:
   - Go to your Trusted Signing account -> Access Control (IAM) -> Add role assignment
   - Role: "Trusted Signing Certificate Profile Signer"
   - Assign to the App Registration created above

**GitHub Secrets Setup:**

Add these 5 secrets to the repository (Settings -> Secrets and variables -> Actions):

| Secret Name | Value Source |
|---|---|
| `AZURE_CLIENT_ID` | App Registration -> Application (client) ID |
| `AZURE_CLIENT_SECRET` | App Registration -> Client secret value |
| `AZURE_TENANT_ID` | App Registration -> Directory (tenant) ID |
| `AZURE_TRUSTED_SIGNING_ACCOUNT_NAME` | Trusted Signing account name |
| `AZURE_CERTIFICATE_PROFILE_NAME` | Certificate profile name |

**Update endpoint if needed:**

If your Trusted Signing account is NOT in West US 2, update the `-e` endpoint URL in `src-tauri/tauri.conf.json` to match your region:
- East US: `https://eus.codesigning.azure.net`
- West US 2: `https://wus2.codesigning.azure.net`
- West Europe: `https://weu.codesigning.azure.net`
- North Europe: `https://neu.codesigning.azure.net`
